## J2 — Foot IK 原理

**正文**:

```
Foot IK = 让脚贴合实际地面高度

基本流程：
  1. 射线检测 — 从脚部向下发射射线，命中地面
  2. 高度差 — 算出动画脚位和实际地面的差距
  3. IK 求解 — 把脚调整到正确高度
  4. 身体补偿 — 脚上抬时骨盆/身体跟着调整

两条腿分别处理：
  左脚着地 → 左脚IK贴合地面
  右脚着地 → 右脚IK贴合地面
  两脚都在空中 → 骨盆整体下移（可选）
```

**配图**: 示意图 — 射线从脚部向下发射，命中地面，标注高度差和IK调整方向

**备注**:
> "Foot IK的思路很直觉——脚在哪里，地面在哪里，把脚拉过去。"射线从脚部骨骼向下打，打到地面就知道高度差。"IK求解就是把脚调整到正确高度。但只有脚动不行——脚被地面抬高了，骨盆也要跟着上移，不然腿会被拉长。"两条腿分别检测、分别调整。

## J3 — UE5实现

**正文**:

```
UE5 中的 Foot IK 实现

1. 射线检测
   从 Foot 骨骼向下发射 Line Trace
   碰撞通道选择地面相关 Channel

2. IK 节点
   Two Bone IK 或 Foot IK 节点
   目标位置 = 射线命中点 + 偏移

3. 骨盆调整
   Pelvis 跟随两只脚中较低的一侧
   防止腿被过度拉伸

4. 参数调节
   IK 权重过渡速度（避免突然跳变）
   射线长度（最远能检测多远）
   最大抬脚高度（超出则不修正）
```

**配图**: 截图 — IK节点配置 + 调试视图下的射线可视化

**备注**:
> "看一下UE里怎么落地。"展示IK节点配置截图，调试视图下可以看到射线从脚部向下打。"关键参数：IK权重过渡要平滑，不然脚会突然跳到新位置；射线长度决定最远能检测多少高度差。"可以展示调试视图：射线可视化+IK目标点可视化，观众能直观看到脚是怎么"找"到地面的。
