## H3 — 解法：Land + Recovery 叠加

**正文**:

```
把落地拆成两层

Land 本体：很短的落地瞬间动画
  只负责脚触地那一刻

Land Recovery：Additive 叠加动画
  表现下坠冲击、身体下沉再恢复
  以 Additive 方式叠加在后续动画之上

好处：
  Land 本体短 → 不影响衔接
  Recovery 是叠加的 → 接 Walk / Jog / Sprint 都行
  资源量可控，不需要乘方向
```

**配图**: 示意图 — Land短动画 + Recovery叠加层 → 后续任意Loop，标注叠加关系

**备注**:
> "关键洞察：把落地拆成两层。"Land本体只管触地瞬间，极短；冲击感由Recovery叠加提供。"Recovery是Additive的，不管接下来是Walk、Jog还是Sprint，Recovery都能给出下坠恢复效果。"跟E阶段的AO类比："AO是叠加做瞄准偏移，Recovery是叠加做落地冲击。都是Additive思路。"

## H4 — 问题二：大跳时 In-Air 太长

**正文**:

```
三段跳的第二个问题：大跳时 In-Air 占时间太长

原地跳：In-Air 0.3 秒，没问题
高处落下：In-Air 可能 2-3 秒，一条动画循环到底

表现问题：
  上升和下降用同一个姿态 → 失去物理感
  长时间单调循环 → 像挂在空中
  没有顶点过渡 → 缺乏节奏变化

需要把 In-Air 拆得更细
```

**配图**: GIF对比 — 短跳In-Air自然 vs 大跳In-Air长时间单调循环，标注时间线

**备注**:
> "落地衔接解决了，但还有一个问题：大跳怎么办？"展示高处落下，In-Air持续2-3秒，角色用同一条动画循环到底。"上升时身体应该后仰，下降时应该前倾，顶点有个过渡——三段跳的In-Air做不到。"引出需要把In-Air拆成更细的阶段。

## H5 — 五段跳跃拆解

**正文**:

```
5 个阶段取代 3 段

Start        → 起跳瞬间（根据起跳速度选择）
Start Loop   → 上升循环（适配不同上升时长）
Apex         → 到达顶点的过渡
Fall Loop    → 下落循环（适配不同下落时长）
Land         → 短落地 + Recovery 叠加

每个阶段的适配方式：
  Start Loop / Fall Loop → 循环控制适配时长
  Apex → 短过渡，衔接上升和下降
  Land → 短动画 + Recovery Additive
```

**配图**: 时间轴图 — 五段跳跃的完整流程，标注每段作用和适配方式

**备注**:
> "In-Air拆成三段：Start Loop（上升循环）、Apex（顶点过渡）、Fall Loop（下落循环）。"Start Loop循环多或少取决于跳多高，Apex是上升转下降的关键过渡，Fall Loop循环多少取决于落多远。"加上前面的Start和后面的Land+Recovery，就是完整的五段跳跃。"跟三段对比："三段是一个In-Air搞定一切，五段让每个阶段各司其职。"

## H6 — UE5实现要点

**正文**:

```
跳跃状态机实现

1. 状态机：5 段状态 + 过渡条件
   Start → 自动 → Start Loop
   Start Loop → 检测到开始下降 → Apex
   Apex → 自动 → Fall Loop
   Fall Loop → 检测落地 → Land

2. 空中循环适配
   Start Loop / Fall Loop 用 Play Rate 或时间缩放
   适配不同跳跃高度带来的时长差异

3. Land + Recovery
   Land 短动画播放
   同时叠加 Recovery Additive
   Recovery 随时间衰减 → 自然恢复到正常姿态

4. 起跳方向
   站立起跳 vs 移动起跳可选用不同 Start 动画
```

**配图**: 截图 — 跳跃状态机 + Recovery叠加节点配置

**备注**:
> "看一下UE里怎么落地。"展示状态机截图，重点是各状态之间的过渡条件。Recovery的叠加展示AnimBP节点图——Apply Additive节点，跟AO同一套机制。"空中循环的适配比较直觉：跳得高就多循环几圈，跳得低就快速通过。Land阶段的Recovery衰减时间可以调参：重落地衰减慢，轻落地衰减快。"
