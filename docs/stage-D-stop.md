# 阶段D：Stop（~12min，10页）

## D1 — 问题：停止时的滑步

**正文**:

```
角色从行走到静止

停止时最容易出现滑步

原因：
  Loop 动画还在播放 → 脚还在迈步
  代码已经停止移动 → 身体停了
  脚和地面之间产生相对滑动

Stop 比 Start 更难
  Start 是从零加速，容错高
  Stop 要精确刹住，误差一目了然

关键参数：减速度（Deceleration）
  减速度决定角色从移动到静止需要多长距离
  直接影响 Stop 动画的选择和截断时机
  高减速度 = 急刹，低减速度 = 缓停
```

**配图**: GIF动图 — 停止瞬间的滑步特写，慢放，圈出脚在地面拖动

**备注**:
> "起步的问题解决了，停下来呢？停止其实比启动更难。"先让观众看滑步特写——停止时脚在地面上多滑了半步，非常明显。对比C阶段的Start："Start是从零加速，稍微差一点看不出来；Stop是要精确刹住，差一帧都看得出来。"重点提一下减速度这个参数："角色的减速度设定会直接影响Stop的实现方式——急刹和缓停是完全不同的场景，后面讲两种方案时会看到区别。"

## D2 — 方式一概述：DM截断式

**正文**:

```
思路：一条 Stop 动画 + Distance Matching 截断

只用一个 Stop 动画
播放时用 DM 位移曲线追踪剩余距离
剩余距离 = 0 时截断播放 → 精确停住

核心逻辑：
  松开摇杆 → 算出还需要走多远才能自然停下
  DM 查曲线 → 找到对应的播放位置
  从那个位置开始播 Stop 动画
```

**配图**: 示意图 — DM曲线图，标注"剩余距离"和"截断点"

**备注**:
> "第一种方式，延续了Start阶段的Distance Matching思路。"回顾C3的DM概念："还记得用距离驱动播放吗？Stop是反过来——算出还剩多少距离要消耗，然后让动画精确播到那个距离停止。"这是DM截断的核心：不是播完整个Stop动画，而是从合适的位置切入、在距离归零时截断。

## D3 — DM截断式实现细节

**正文**:

```
实现要点

状态机：Loop → Stop 状态切换
触发条件：松开摇杆 / 速度降为 0

DM 截断流程：
  1. 记录当前速度和已走距离
  2. 计算减速到 0 需要的距离
  3. 在 Stop 动画的 DM 曲线上找到对应时间点
  4. 从该点开始播放，距离归零时停止

动画资源要求：
  一条足够长的 Stop 动画
  保留 Root Motion → 烘焙 DM 曲线
```

**配图**: 截图 — 状态机逻辑图 + DM曲线可视化，标注截断机制

**备注**:
> "看一下具体怎么落地。"展示状态机截图和DM曲线编辑器截图。关键让观众理解"截断"：Stop动画本身可能是1秒，但DM可能只播0.4秒就截断了，因为距离已经消耗完了。不需要看代码，流程图+截图够用。可以提一句："这个方案只需要一条Stop动画，资源量最小。"

## D5 — 方式二概述：左右脚选择式

**正文**:

```
思路：检测着地脚 → 选择对应 Stop 动画

问题：
  停止瞬间，左脚着地和右脚着地
  需要不同的 Stop 动画才能自然收脚

做法：
  准备两条 Stop 动画（左脚停 / 右脚停）
  检测当前哪只脚着地
  选对应的动画播放
```

**配图**: 示意图 — 左脚着地瞬间 vs 右脚着地瞬间的角色截图，标注不同的收脚姿态

**备注**:
> "第二种方式，思路完全不同——不是截断动画，而是选择正确的动画。"直观解释："你走路时突然停下来，左脚在前和右脚在前，收脚的姿势是不一样的。所以准备两条动画，运行时选。"这个方案对美术要求更高（两条动画），但效果可能更自然。

## D6 — 左右脚选择式实现细节

**正文**:

```
实现要点

脚部检测逻辑：
  追踪双脚 IK 目标点的 Y 轴高度
  高度最低的脚 = 当前着地脚
  或用动画曲线标记着地帧

动画选择：
  左脚着地 → 播左脚 Stop
  右脚着地 → 播右脚 Stop

过渡处理：
  Loop → Stop 的 Blending 时间要短
  避免检测到播放之间脚部状态变化
```

**配图**: 截图 — 脚部高度检测示意图 + 动画曲线标注着地帧

**备注**:
> "关键难点是检测时机——必须在切换动画的那一刻判断哪只脚着地，不能延迟。"展示检测逻辑的示意图。可以用角色骨骼的脚部IK目标做可视化：着地的脚标绿色，悬空的脚标红色。动画曲线方案更精确：在动画上标记着地帧，查曲线比算IK高度更可靠。

## D7 — 锁脚 IK 机制

**正文**:

```
左右脚方案的辅助：锁脚 IK

即使选对了 Stop 动画
仍可能出现微小滑步

锁脚 IK：
  停止瞬间锁定着地脚的世界位置
  IK 将脚钉在原地，不随身体继续滑动
  身体稳定后释放 IK

本质：IK 补偿动画的最后一小段误差
```

**配图**: GIF动图 — 锁脚IK ON/OFF对比，圈出脚被锁定在原地

**备注**:
> "选对了动画也不够完美，因为动画和实际停止位置之间总有微小误差。"锁脚IK的效果：脚在停止瞬间被IK钉在地面位置，身体继续减速到停稳后释放。这是"用IK兜底"的思路——动画做到90%，剩下10%用IK补。跟B14的Stride Warping做类比："一个改步幅防滑步，一个锁脚位防滑步，都是程序补动画的思路。"

## D8 — 两种方式对比

**正文**:

```
            DM截断式          左右脚选择式
动画资源      1条 Stop         2条 Stop（左/右）
核心机制     距离截断          脚部检测 + 选择
辅助机制     无               锁脚 IK
资源成本      低               中

适用场景：
  缓停（低减速度/有刹车距离）→ DM截断式更稳
    刹车距离长，DM 有足够窗口精确截断

  急刹（高减速度/瞬间停）→ 左右脚式稍适合
    急刹时 DM 截断窗口太小，表现力有限
    左右脚选择 + 锁脚 IK 更可控

实际项目常混合使用
```

**配图**: 表格 + 两种方式在不同减速度下的效果对比

**备注**:
> "两种方式不是简单的优劣关系，而是适用于不同场景。"回到D1提的减速度概念："缓停时DM截断有足够的距离窗口做精确截断，效果很好；急刹时距离窗口极短，DM截断来不及表现，左右脚选择+锁脚IK更可控。"很多项目会混合：根据减速度或速度阈值动态切换方案。问互动："你们项目角色是急刹多还是缓停多？"
