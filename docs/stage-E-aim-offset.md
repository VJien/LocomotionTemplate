# 阶段E：Aim Offset（~6min，5页）

## E1 — 问题：没有上半身跟随

**正文**:

```
角色能走能停，但上半身是"死的"

没有 Aim Offset 时：
  上半身始终朝前方
  瞄准方向变了，身体不跟着动
  像一块木板在移动
```

**配图**: GIF动图 — 角色转身瞄准时上半身僵硬不动，圈出不跟随的区域

**备注**:
> "走路和停止都解决了，但角色看起来还是不对——上半身像焊死了一样。"展示角色转动视角时上半身完全不动、只有头和手微调的表现。这是第三人称射击/动作游戏最基础的需求。"这个问题用Aim Offset解决。"

## E2 — Aim Offset 原理

**正文**:

```
Aim Offset = 叠加式偏移动画

思路：
  上下半身分离处理
  下半身：负责移动（Loop/Start/Stop）
  上半身：负责瞄准方向（Aim Offset）

实现：
  制作一组 Additive 动画（偏移量）
  根据瞄准的 Pitch / Yaw 值混合
  叠加到基础动画之上

关键：Additive 模式
  不是替换动画，是在原有动画上叠加偏移
```

**配图**: 示意图 — 角色分层图（下半身移动层 + 上半身AO层叠加）；AO二维网格（Pitch × Yaw）及动画分布

**备注**:
> "核心思路是上下半身分而治之。"下半身继续用Loop/Start/Stop系统，上半身单独用AO做瞄准偏移。重点解释Additive："不是播一段新动画替换旧的，而是在旧动画上面叠加一个偏移量——身体本来在走，AO只是让上半身额外偏一点。"用类比："就像你在走路的同时转头看路牌——走路是下半身的事，转头是上半身的事，两件事同时做互不干扰。"

## E3 — UE5实现

**正文**:

```
UE5 中的 Aim Offset 工作流

1. 制作 AO 资产
   录制各方向瞄准姿势
   转为 Additive 动画
   组织成 Aim Offset 或 Aim Offset Blend Space

2. AnimBP 中配置
   获取控制器 Pitch / Yaw
   映射到 AO 的输入参数
   用 Apply Additive 节点叠加到基础动画
```

**配图**: 截图 — AO编辑器 + AnimBP中的节点连接图

**备注**:
> "原理讲了，UE里怎么做？"展示AO资产编辑器截图，让观众看到姿势是如何分布在Pitch/Yaw二维空间中的。然后展示AnimBP节点图——关键是Apply Additive节点的位置，它叠加在Layered Blend per Bone之后。"美术需要知道：AO的姿势数量和覆盖范围直接决定瞄准的自然度，角度覆盖不够会出现死区。"

## E4 — 坑点：混合中间角度的穿模

**正文**:

```
AO 资源的隐藏坑：混合出来的中间角度

资源制作了 Up 和 Center 两个姿势
单独看都没问题

但混合出来的中间角度（如 45° 斜上方）：
  手臂可能太靠近身体
  武器穿模/手臂扭曲

例子：弓箭挂左手武器点
  Up 姿势：左手位置正确
  Center 姿势：左手位置正确
  混合 45°：左手被拉到身体内侧 → 弓穿模

本质：Additive 混合是骨骼空间插值
  不保证末端（手/武器）的世界位置合理
```

**配图**: 截图/动图 — Up姿势OK + Center姿势OK + 混合中间角度时武器穿模/手臂扭曲的特写

**备注**:
> "AO看起来简单，但实际制作中有个很常见的坑。"展示具体例子：角色拿弓箭，Up和Center单独看都完美，但混合到中间角度时弓穿进身体里。解释原因："Additive混合是在骨骼空间做插值，只保证骨骼角度在中间，不保证手在世界空间的位置合理。"这不是bug，是方案本身的局限。解决思路：增加采样姿势（不只是Up/Center，补几个中间角度的姿势），或者在特定角度加IK修正。给美术的建议："做AO的时候，不要只看极端姿势，一定要逐角度检查混合结果，尤其是拿武器的状态。"
