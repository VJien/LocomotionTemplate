## C2 — 两种 Start 资源风格

**正文**:

```
风格一：长动画衔接（如 Paragon 资源）

  Start 后半段 ≈ 一个完整 Loop
  加速过程 + 行走循环 包含在一条动画里
  优点：过渡自然，接 Loop 无缝
  缺点：单条动画长，资源体积大

风格二：短动画直连（如 COD 资源）

  Start 很短（0.3-0.6 秒）
  最后一帧严格匹配 Loop 第一帧
  优点：紧凑，资源小
  缺点：姿势匹配要求高，过渡点需要精确

共同要求：
  保留 Root Motion 数据
  结尾姿势与 Loop 衔接
```

**配图**: 上下两组帧序列对比 — 上组长动画（Paragon）展示后半段融入Loop，下组短动画（COD）展示精确帧匹配

**备注**:
> "Start动画有两种常见风格，各有取舍。"用实际资源举例：Paragon的长Start，后半段已经是在走了，接Loop基本无感；COD的短Start，像踢一脚把角色踹进走的状态，要求最后一帧跟Loop第一帧严丝合缝。给美术的建议："长风格适合动作感强的角色，短风格适合快节奏游戏。选哪种取决于项目调性。"

## C3 — Start 阶段的滑步与 Distance Matching

**正文**:

```
有了 Start 动画，还是会滑步

动画有自己的位移节奏
代码按速度驱动物体移动
两者不同步 → 起步瞬间滑步

Distance Matching 的思路：
  不用时间驱动播放，用距离驱动

记录动画每帧的累计位移 → 位移曲线
代码算出角色已经走了多远 → 查曲线得到播放时间点
动画位移 = 代码位移 → 不滑步
```

**配图**: 曲线图 — 横轴时间/距离，纵轴累计位移，标注"时间驱动 vs 距离驱动"的区别；Start动画中滑步瞬间标注

**备注**:
> "上一阶段讲了Loop的滑步用Stride Warping解决，但Start阶段的滑步性质不同——它发生在从静止到移动的过渡瞬间，Stride Warping帮不了。"衔接B14："还记得Stride Warping是改步幅吗？Start的时候角色还没走起来，没有步幅可改。"然后引出DM："Distance Matching换了一个思路——不是改动画，而是改播放方式：用距离驱动而不是时间驱动。"关键是"用距离查时间"这个反转——不是"到了第几秒播第几帧"，而是"角色走了多远→查曲线→播到对应帧"。

## C4 — UE5实现：Curve烘焙与DM节点

**正文**:

```
UE5 中的 Distance Matching 工作流

1. 动画导入时保留 Root Motion
2. Animation Modifier 烘焙位移曲线
   → 生成 Distance Curve
3. AnimNode 中使用曲线驱动播放位置
   → 动画播放由距离决定，不由时间决定

关键节点：
  Distance Matching 节点
  可在 Start / Stop 两个阶段使用
```

**配图**: 截图 — Animation Modifier配置 + Distance Curve在编辑器中的可视化

**备注**:
> "原理讲了，UE里怎么落地？"展示实际工作流截图。Animation Modifier是自动化工具，批量给动画烘焙位移曲线，不需要手动画。然后展示曲线长什么样——一条单调递增的曲线，横轴是帧，纵轴是累计距离。DM节点读取这条曲线，用代码算出的目标距离反查播放位置。不需要深入代码层面，截图+流程图够用。
