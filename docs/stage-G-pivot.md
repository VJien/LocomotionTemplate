  播放专用的转向过渡动画
  脚从旧方向自然切换到新方向

## G2 — Pivot 概念与触发逻辑

**正文**:

```
Pivot = 运动中方向突变的动画过渡

触发条件：
  角色处于移动状态（Loop）
  移动方向发生接近 180° 的反转
  速度未降为零（降为零是 Stop，不是 Pivot）

Pivot 动画的作用：
  桥接旧方向 Loop → 新方向 Loop
  展现减速 → 转身 → 加速的自然过程

方向分类：
  接近 180° 反转 → 最常见，需求最强
  90° 转向 → 可选覆盖
  小角度 → 不需要 Pivot，Blend Space 自然过渡
```

**配图**: 示意图 — 角色运动轨迹，标注方向突变点和Pivot动画覆盖区间

**备注**:
> "Pivot跟Stop的区别：Stop是速度归零，Pivot是速度还在但方向变了。"最典型的场景：角色往前跑，玩家突然把摇杆拉到反方向——Loop直接切到反方向会很生硬，Pivot动画提供自然的转身过渡。不需要所有角度都有Pivot动画，180°最重要，90°次之。

## G3 — Pivot 实现

**正文**:

```
UE5 中的 Pivot 工作流

1. Pivot 动画资源
   180° 转身动画（左转/右转）
   可选 90° 转身动画
   保留 Root Motion 数据

2. 触发逻辑
   检测移动方向变化量
   变化超过阈值 → 触发 Pivot
   根据方向变化选择对应动画

3. 与 DM 的配合
   Pivot 动画同样可用 Distance Matching
   确保位移过渡平滑，不滑步
```

**配图**: 截图 — Pivot触发逻辑示意 + 状态机中Pivot状态的位置

**备注**:
> "跟Start/Stop类似，Pivot也需要动画和代码的配合。"展示状态机中Pivot的位置——它在Loop状态之间，作为方向切换的桥梁。可以提一下DM的复用："之前Start/Stop用DM做距离匹配，Pivot也可以用同样的机制保证位移平滑。"
