# 阶段K：前沿技术（~8min，5页）

## K1 — 传统方案的天花板

**正文**:

```
我们今天搭的系统长这样

状态机 + Blend Space + DM + IK + AO + ……

每加一个行为 → 多一套状态/动画/过渡逻辑
行为越多 → 状态机越膨胀 → 维护成本爆炸

瓶颈：
  动画资源量随行为数线性增长
  状态机复杂度随行为数指数增长
  每个行为都要手工调参 → 不可扩展
```

**配图**: 示意图 — 复杂状态机网络图（节点密密麻麻的蜘蛛网）

**备注**:
> "我们今天从Loop一路搭到Foot IK，系统已经相当完整了。"回顾脑图（A4），每加一个功能就多一套状态、动画、过渡。"这种方案到一定程度就会撞墙——行为越多，状态机越复杂，美术做的动画越多，程序写的逻辑越绕。有没有另一种思路？"

## K2 — Motion Matching 原理

**正文**:

```
Motion Matching：数据驱动，不用状态机

思路反转：
  传统：手工设计状态机 → 选择动画
  MM：每帧从动画数据库中搜索最匹配的片段

工作流程：
  1. 录制大量高质量动画（跑步、转向、急停、跳跃……）
  2. 提取每帧的特征（位置、速度、朝向、姿态……）
  3. 运行时：根据当前状态和目标，搜索特征最接近的帧
  4. 跳转到该帧，继续播放

核心优势：
  不需要手写状态机
  不需要手工设计过渡
  行为越丰富效果越好
```

**配图**: 示意图 — 特征空间搜索可视化（当前点 vs 候选动画帧分布）

**备注**:
> "Motion Matching的思路完全不同。"传统方案是"设计师决定什么时候播什么动画"，MM是"每帧从数据库里找最合适的动画"。"录大量动画，提取特征，运行时搜索匹配——没有状态机，没有过渡设计，算法自动找最佳帧。"类比："传统方案是按食谱做菜，MM是自助餐——你想吃什么，系统自动给你找最像的。"

## K4 — 其他前沿方向

**正文**:

```
Motion Matching 之外，还有更多

ML Deformer
  用机器学习做形变修正
  肌肉、布料、软体的实时形变

Neural Animation
  神经网络驱动的动画生成
  学习运动模式，生成新动画

Learned Motion Matching
  MM + 神经网络
  用网络替代暴力搜索，降低运行时开销

Full Body IK in Motion
  全身 IK 在运动中实时求解
  替代预设动画的思路
```

**备注**:
> 快速过一遍前沿方向，不需要深入。重点是让听众知道"业界还在往前走"。ML Deformer和Neural Animation是学术界和工业界都在探索的方向。"这些技术大多还在早期，但MM已经开始在商业项目中落地了。"
