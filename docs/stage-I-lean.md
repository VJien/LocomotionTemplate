# 阶段I：Lean（~6min，5页）

## I1 — Lean 是什么

**正文**:

```
Lean = 角色转向时的身体倾斜

生活中的例子：
  骑车转弯 → 身体侧倾
  跑步过弯 → 重心内倾
  滑雪转弯 → 整个人压过去

游戏里：
  没有 Lean → 角色像直杆一样转向
  有了 Lean → 身体朝转向方向倾斜，有重心感
```

**配图**: 生活照片（骑车转弯侧倾） + GIF对比（游戏角色无Lean笔直 vs 有Lean倾斜）

**备注**:
> "先说Lean是什么，因为很多人没听过这个词。"用生活例子建立直觉：骑车转弯、跑步过弯、滑雪转弯——身体都会朝转向方向倾斜。"游戏里的Lean就是模拟这个——让角色转向时身体自然倾斜，而不是像一根棍子一样直挺挺地转。"

## I2 — Lean 原理

**正文**:

```
Lean = 基于角速度/加速度的身体倾斜

触发逻辑：
  追踪角色的角速度（朝向变化速率）
  角速度大 → 倾斜角度大
  角速度小 → 倾斜角度小
  角速度归零 → 回正

实现方式：
  从动画中提取 Lean 曲线（Yaw Rate → Roll 偏移）
  或实时计算角速度 → 映射到 Roll 轴偏移
  叠加到角色骨骼上

本质：让物理行为在视觉上有反馈
```

**配图**: 示意图 — 角速度→倾斜角度映射曲线 + 角色不同转向速度下的倾斜姿态

**备注**:
> "Lean的核心是：角速度驱动身体倾斜。"角速度大说明角色在急转，身体应该倾斜更多。"这不是动画资源的问题，而是程序根据物理参数实时计算的视觉反馈。"可以类比骑车或滑雪转弯——速度越快、弯越急，身体倾斜越明显。

## I3 — UE5实现

**正文**:

```
UE5 中的 Lean 实现

方式一：动画曲线驱动
  动画资源中烘焙 Lean 曲线（Yaw Rate → Roll）
  运行时读取曲线值 → Apply Roll 到骨骼

方式二：实时计算
  追踪 Actor Yaw 变化 → 计算角速度
  映射函数：角速度 → Roll 偏移量
  叠加到 Spine / Pelvis 骨骼

参数调节：
  倾斜灵敏度（角速度→角度的映射比例）
  倾斜速度（响应快慢）
  最大倾斜角度
```

**配图**: 截图 — Lean曲线编辑器 + AnimBP中的节点配置

**备注**:
> "两种方式都可以，曲线驱动更精确（动画本身带信息），实时计算更灵活（不依赖动画数据）。"展示曲线和节点配置截图。参数调节是关键："倾斜太灵敏→角色看起来晃来晃去，太迟钝→感觉不到效果。需要根据项目调性找到平衡点。"
