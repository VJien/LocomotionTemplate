# 阶段F：Turn In Place（~8min，6页）

## F1 — 问题：原地转向不自然

**正文**:

```
角色站着不动时转动视角

没有 Turn In Place 时：
  角色整体硬转（Snap）
  脚在地面打滑旋转
  或者完全不转，只有头/上半身动

自然表现应该是：
  脚一步一步地转过来
  身体跟随视角逐帧调整
```

**配图**: GIF动图 — 角色原地硬转（脚滑/身体整体旋转），圈出问题

**备注**:
> "上半身能跟随瞄准了，但角色站定不动时怎么转身？"展示硬转的表现——角色像陀螺一样整体旋转，脚在地面打滑。或者另一种极端：只有头转，身体不动，像猫头鹰。"自然的转身应该是一步一步转过来的——这就是Turn In Place。"

## F2 — Turn In Place 原理：与 AO 协同工作

**正文**:

```
TIP 不是独立的转向动画，而是与 Aim Offset 深度协同

完整流程：

1. 玩家转动相机
   → AO Yaw 累积在上半身
   → 上半身跟随相机方向偏转
   → 下半身保持不动（通过反向 AO Yaw 抵消，保持站定）

2. 偏差角逐渐增大
   → 上半身扭得越来越多
   → 下半身始终朝原方向

3. 偏差超过阈值 → 触发 TIP
   → 播放转向动画，整个角色底座旋转追赶相机
   → TIP 完成后，基础朝向更新
   → AO Yaw 归零（底座追上了，偏差消失）
```

**配图**: 示意图 — 三阶段角色状态图：小偏差（上半身微扭）/ 中偏差（上半身明显扭转）/ TIP触发（底座旋转追赶）

**备注**:
> "TIP不能脱离AO单独理解。"先回顾E阶段的AO："上半身通过AO跟随相机方向。"然后引入TIP："下半身怎么办？下半身不跟——通过反向AO Yaw让它保持朝前。随着相机继续转，上半身扭得越来越厉害，当偏差超过阈值，就触发TIP让整个角色底座旋转追上来。"核心概念：TIP和AO是一个连续体，AO负责实时微调，TIP负责大角度重定向。TIP完成后AO Yaw归零，角色朝向和相机重新对齐。

## F3 — TIP与相机/输入的关系

**正文**:

```
TIP 的触发与相机系统紧密耦合

触发条件：
  角色处于 Idle 状态
  视角方向与身体朝向偏差 > 阈值

不触发的情况：
  角色在移动中（转向由移动系统处理）
  偏差在阈值内（小角度用 AO 吸收）

相机跟随：
  TIP 播放期间相机保持跟随
  朝向更新后相机同步
  避免相机和角色不同步的抖动
```

**配图**: 示意图 — 角色朝向 vs 相机朝向 vs TIP触发区域的角度关系图

**备注**:
> "TIP不只是动画的问题，它跟相机和输入系统都有关联。"解释为什么只在Idle时触发："移动中的转向是移动系统的事（Orient Rotation to Movement），不需要TIP。"相机同步也很关键："TIP播完朝向变了，相机要跟过来，不然玩家会觉得视角跳了一下。"这块是程序需要注意的细节。

## F4 — UE5实现

**正文**:

```
UE5 中的 TIP 工作流

1. TIP 动画资源
   左转/右转各若干条（90° / 180° 等）
   带 Root Motion 旋转

2. AnimBP 逻辑
   Idle 状态下检测视角偏差
   偏差超阈值 → 选对应角度的 TIP 动画
   播放完毕 → 更新 Actor 朝向

3. 可选优化
   小角度：AO 吸收，不触发 TIP
   中角度：短 TIP 动画
   大角度：长 TIP 动画或多次触发
```

**配图**: 截图 — AnimBP中的TIP状态逻辑 + 角度阈值配置

**备注**:
> "看一下UE里怎么落地。"展示AnimBP截图，重点是Idle状态下TIP的分支逻辑。可以展示角度阈值的配置方式——通常用曲线或表格映射偏差角度到TIP动画选择。"资源方面，角度越细分转得越自然，但资源量也越大。"
